The SHAFT docs repo is a SEPARATE repo (default branch `master`, PRs squash-merged). Merge policy: `master` requires 1 approving review, but `enforce_admins` is OFF and owner MohabMohie is in `bypass_pull_request_allowances`, so the owner merges own PRs with `gh pr merge <n> --squash --admin --delete-branch` (a plain `--squash` returns 'base branch policy prohibits the merge' / mergeStateStatus BLOCKED). This is the sanctioned solo-owner path. Deploy checks: each PR runs Netlify `deploy-preview` + Cloudflare `Workers Builds` (both run `docusaurus build`); 'Header rules / Pages changed' showing skipping is normal. There is no npm-test CI gate -- run docs tests locally (`npm run test:docs`, `test:homepage`, `typecheck`). Build fragility: bare `<url>`/`<path>`-style placeholder tokens in `.md` prose fail the Docusaurus MDX build (unclosed-JSX-tag error), silently red-ing the deploy preview until escaped (`\<url\>`); always run `npx docusaurus build` locally and check the REAL exit code (`cmd | tail` masks it) before merging. Assertion note: SHAFT assertions execute immediately on their terminal call; `.perform()` is a redundant no-op. `matchesScreenshot()` used to be the exception (fixed in SHAFT_ENGINE#3524 to take a `VisualComparisonOptions` arg and run immediately).
