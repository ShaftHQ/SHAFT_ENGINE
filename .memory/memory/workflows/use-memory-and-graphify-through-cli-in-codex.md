For SHAFT_ENGINE, use the CLI from the repository root. Read Memory with `memory load "task summary"` for task context, `memory search "query"` for targeted lookup, and `memory inspect <id>` for details; do not try `memory query`, because this installed CLI does not provide that subcommand. Write durable Memory with `memory remember --stdin` using intent-first JSON, then run `memory diff` to review; use `memory save --stdin` only for advanced structured patches. For Graphify, prefer `graphify query "question"` when `graphify-out/graph.json` exists, and `graphify update .` after code/relationship changes. If `gh run download` fails on unarchived artifacts, use GitHub job logs/API instead of forcing artifact extraction. Use CLI first; MCP only when Memory/Graphify tools are actually exposed in the current tool list.
