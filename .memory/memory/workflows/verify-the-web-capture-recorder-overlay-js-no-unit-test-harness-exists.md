The web Capture recorder overlay is browser-injected JS at `shaft-capture/src/main/resources/browser/shaft-capture-recorder.js` -- a single arrow-factory IIFE `(channel, sink) => {...}` (~2500 lines, CSS+DOM inlined). No JS unit test / Node DOM-stub harness exists; the only committed test is the real-browser `ManagedCaptureRecorderBrowserTest` (`@Tag("external-e2e")`, run via shaft-capture-e2e.yml / `-DincludeCaptureBrowserE2E`). Recipe: (1) syntax-check every edit -- `node -e "const fs=require('fs'),vm=require('vm'); new vm.Script('('+fs.readFileSync('<path>','utf8')+')')"` (a syntax error breaks the WHOLE overlay); (2) real-execution smoke via headless Chrome -- serve a harness dir (`py -3 -m http.server`), harness HTML does `fetch('recorder.js').then(t=>{ const f=eval('('+t+')'); f(null,null); })` to mount, then drives + asserts into a `#results` div; run `chrome --headless=new --no-sandbox --virtual-time-budget=4000 --dump-dom <url>` and grep the DOM for FAIL; (3) add a committed assertion to `ManagedCaptureRecorderBrowserTest`. Gotcha: the overlay records the initial page as an 'Open ...' breadcrumb immediately, so a fresh recording's `uiState.actions.length` is 1, never 0 -- don't gate first-run coach marks on `actions.length === 0`; gate on the session-scoped dismissed flag. Only the top frame owns the overlay (`topLevel` gate); state persists in `sessionStorage` key `shaft.capture.recorder.ui`; the seam `globalThis.__shaftCaptureUiState` is exposed for black-box driving.
