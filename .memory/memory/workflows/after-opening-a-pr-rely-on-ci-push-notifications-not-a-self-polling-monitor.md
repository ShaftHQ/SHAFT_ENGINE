Workflow correction (user feedback 2026-07-14): after opening a GitHub PR, do NOT arm a self-polling `gh pr checks` background Monitor to wait for CI. It idles as a frozen background agent, wasting a task slot and wall-clock time. This environment already PUSHES <ci-monitor-event> notifications when a CI check fails or a review comment (e.g. Codacy) lands on the PR. Rely on those to wake up, then run `gh pr checks` once and merge if all green or fix if failed. Use only a long ScheduleWakeup (~20-30 min) as a safety-net fallback heartbeat, never a tight polling monitor.
