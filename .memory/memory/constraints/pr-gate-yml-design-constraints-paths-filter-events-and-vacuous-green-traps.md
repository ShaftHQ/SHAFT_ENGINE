dorny/paths-filter cannot resolve a diff base outside push/pull_request events, so pr-gate.yml is deliberately pull_request-only (a workflow_dispatch run would fail the changes job or pass vacuously). GitHub applies an implicit success() to dependent jobs whose if-condition lacks a status function: if the changes job is skipped, every leg is skipped and a naive summary goes green having tested nothing. The summary job must explicitly fail unless needs.changes.result == 'success' (fail-closed). Required check context on main: 'PR Gate Summary' (ruleset id 19182579).
