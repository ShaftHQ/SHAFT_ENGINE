At the start of every session, before substantive work: run `git worktree list`; never remove or modify the current session's own worktree; for any OTHER worktree, only remove it after positively confirming it's safe (created by a prior run you recognize, its branch already merged/deleted on the remote, and `git status --porcelain` shows nothing not already superseded by a merged commit). This repo is used by multiple concurrent Claude Code sessions -- an unfamiliar worktree may belong to another session working right now; never assume it's stale garbage, leave it alone if in doubt, ask the user rather than removing it. After cleanup the session should work from a checkout current with remote main -- if the session's own worktree is stale, sync it rather than fabricating a new one. For bulk branch/worktree deletion: a branch is safe to delete if `git merge-base --is-ancestor <branch> origin/main` is true, or (after a squash-merge changes the commit hash) if its tip commit message matches one in `git log origin/main --oneline --grep="<message>" -F`; directories with no `.git` and zero files are always safe to remove regardless. Only delete when one of these checks passes; otherwise leave it and ask.
