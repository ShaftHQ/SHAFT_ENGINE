User correction (2026-07-17): when opening a PR via `gh pr create`, the body must use a GitHub closing keyword ("Closes #NNNN", one per issue) for every issue the PR resolves -- not just a plain reference like "#NNNN" or a bullet list -- so the issue auto-closes on merge instead of staying open. Caught after PR #3654 was opened with plain '#3634 ...' bullets instead of 'Closes #3634'; fixed via `gh pr edit` before merge. How to apply: whenever drafting a PR body (work-github playbook close-out step, or any ad hoc PR), use 'Closes #N' (or 'Fixes #N') for each issue fully addressed by that PR; for partial/umbrella coverage, only close the sub-issues actually shipped in that PR, not the umbrella issue itself unless the PR closes it entirely.
