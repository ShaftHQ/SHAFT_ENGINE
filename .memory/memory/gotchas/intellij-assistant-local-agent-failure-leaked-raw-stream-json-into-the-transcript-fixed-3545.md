A failed structured local-agent run (Claude stream-json / Codex --json) in the IntelliJ SHAFT Assistant used to fall back to raw stdout, so native NDJSON events like {"type":"system","subtype":"thinking_tokens",...} leaked verbatim into the transcript (multi-object NDJSON also defeats AssistantMarkdown.normalizeMarkdown's single-value JSON parse, rendering unformatted). Root cause: AssistantLocalAgentRunner.run() used streamParser.finalOutput() only on success. Fix (PR #3545): on structured-command failure render StructuredStreamParser.failureOutput() -- terminal error answer text, else a plain-language reason (error subtype / turn.failed detail / exit code) + stderr; raw NDJSON never surfaced. Non-structured/custom commands keep raw-stdout passthrough (plain prose). Note: MCP-tool (non-agent) failures were ALREADY humanized via AssistantMarkdown.toolFailureMarkdown (#3513 S3) -- the leak was local-agent-only. Regression tests in AssistantLocalAgentRunnerVerboseStreamTest.
