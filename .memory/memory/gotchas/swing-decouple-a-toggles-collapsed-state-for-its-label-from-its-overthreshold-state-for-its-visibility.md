In AssistantTranscriptView's CollapsibleOutputPanel (#3629), the first implementation computed `collapsed = overThreshold && !userExpanded`. For short/under-threshold content (overThreshold=false, toggle hidden via setVisible(false)) this forced collapsed=false, so the hidden toggle silently carried the wrong default label ("Hide full output" instead of "Show full output"). It only surfaced once that same bubble later grew past the threshold via a streamed replaceLast() and the toggle became visible for the first time -- a caught-by-under-threshold-test bug, not by the over-threshold happy path. Fix: keep the two booleans independent -- `collapsed = !userExpanded` (drives the label/clip height, correct even while hidden) and `toggle.setVisible(overThreshold)` (drives whether it's shown at all) as separate assignments. General pattern: when a toggle can be present-but-invisible, its label/state logic must be correct in that invisible state too, not just once it becomes visible.
