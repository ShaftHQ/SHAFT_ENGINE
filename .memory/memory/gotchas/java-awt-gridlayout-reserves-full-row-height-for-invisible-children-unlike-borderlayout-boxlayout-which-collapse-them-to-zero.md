ShaftAssistantPanel's constructor built the setup/fresh-project banner strip ('notices') with `new JPanel(new GridLayout(0, 1))`. Both banner panels are frequently `setVisible(false)` (MCP already configured, project not 'fresh'), but GridLayout.preferredLayoutSize()/layoutContainer() iterate ALL child components via parent.getComponentCount() with no isVisible() check, unlike BorderLayout (whose getChild() explicitly filters out invisible children) or BoxLayout (whose SizeRequirements calculation zeroes out invisible children). With both banners hidden this reserved ~64px of pure dead space above the chat header, reported as issue #3694's 'large empty vertical gap'. Fixed in PR branch ChaosEngine/3694-3696-assistant-panel-layout-and-approval-colors by switching to `new BoxLayout(notices, BoxLayout.Y_AXIS)`. Grep for `new GridLayout` before reusing it anywhere a child's visibility is conditional -- other GridLayout call sites exist in shaft-intellij (VisualBaselinesPanel.java, ShaftIcons.java, GuidedWorkflowPanel.java, ShaftRunConfigurationOverridesPanel.java) that were not audited in this pass and may carry the same latent defect if any of their children are ever made conditionally invisible.
