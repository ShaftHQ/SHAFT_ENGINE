`.memory/events.jsonl` is an append-only JSON-Lines event log -- every `memory remember`/`memory save` call appends new lines to it. Since this repo is worked on by multiple concurrent Claude Code sessions/worktrees (see constraint.worktree-housekeeping-at-session-start), two branches that both wrote memory events since a common ancestor will conflict in this exact file when one is merged/rebased onto the other, even though neither branch's actual intent conflicts. Resolution is mechanical and safe: open the conflict markers, keep BOTH sides' appended lines (never drop either), and interleave them by their `timestamp` field for readability -- order does not affect correctness, `memory rebuild`/`memory check` only care that every line is valid JSON, not the ordering. After resolving, always run `memory check` then `memory rebuild` to confirm the index still reflects the merged canonical objects before committing the merge. This is expected, not a sign anything is actually broken -- do not resolve it by dropping one side's events.
