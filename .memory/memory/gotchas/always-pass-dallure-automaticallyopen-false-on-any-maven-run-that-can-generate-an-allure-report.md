SHAFT.Properties.allure.automaticallyOpen (shaft-engine/src/main/java/com/shaft/properties/internal/Allure.java, property key allure.automaticallyOpen) defaults to true and controls whether SHAFT opens the generated Allure HTML report in the default browser after a test run completes. This is a GUI-open action and must never fire in an agentic/headless session -- pass -Dallure.automaticallyOpen=false on every Maven invocation that can reach report generation, in addition to -DheadlessExecution=true. Do not confuse this with allure.open (the Allure-3-CLI-native 'open' option written to allurerc.yaml, controlling whether the allure CLI itself opens the report) -- that one already defaults to false and is a separate, already-safe property. AGENTS.md's Windows/Codex Safety section previously said only the vague 'Maven: add GUI-off Allure/Lighthouse -D...=false'; it now names allure.automaticallyOpen explicitly as a hard requirement.
