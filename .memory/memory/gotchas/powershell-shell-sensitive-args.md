PowerShell parses `@`, braces, semicolons, ampersands, pipes, and Maven-style `-D` values in ways that can split or reinterpret arguments. Before running commands in SHAFT on PowerShell, quote shell-sensitive arguments up front: Maven properties as `'-Dname=value'`, Git stash refs as `'stash@{0}'`, and any argument containing `{}`, `@`, `;`, `&`, or `|`. Evidence: the maintainer corrected repeated command failures from unquoted Maven `-D` arguments and an unquoted `stash@{0}` ref on 2026-06-17.
