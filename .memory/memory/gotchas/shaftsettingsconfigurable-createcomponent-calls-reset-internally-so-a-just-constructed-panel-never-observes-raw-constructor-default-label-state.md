shaft-intellij's ShaftSettingsConfigurable.createComponent() (line ~324) calls reset() before returning the built JPanel. Any label whose constructor helper sets a placeholder value later overwritten by reset()'s real state -- e.g. keyStatusLabel(providerName) constructs a JLabel with text "Checking..." -- can never actually be observed at that placeholder value by a caller holding the panel returned from createComponent(); by the time the caller can inspect it, reset() has already replaced it with the real computed state (e.g. "No stored key."/"Stored in Password Safe." from updateStoredKeyStatus()). A test asserting the constructor's literal placeholder text/description will fail with a confusing mismatch, not because the mirroring logic is wrong, but because that intermediate state is unreachable from outside the class. Fix/workaround: assert against the first real, post-reset() state instead, then prove live-tracking by mutating credentials/settings and calling reset() again to observe a second, genuinely different value.
