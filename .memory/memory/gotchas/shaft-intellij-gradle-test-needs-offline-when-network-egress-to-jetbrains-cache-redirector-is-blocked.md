The IntelliJ Platform Gradle plugin's instrumentTestCode task tries to resolve com.jetbrains.intellij.java:java-compiler-ant-tasks from https://cache-redirector.jetbrains.com on every `gradle test` invocation unless the artifact is already cached, even though `gradle compileJava`/`compileTestJava` alone never need it. In a sandboxed environment with blocked network egress to that host, `gradle test` fails with a socket connect timeout during dependency resolution. Passing --offline works when the artifact is already present in the Gradle cache (it was, on this machine) and skips the redundant metadata check entirely, without needing to touch network. If a fresh environment has never resolved that artifact before, --offline alone will not fix a genuinely cold cache -- compileJava/compileTestJava still work standalone as a fallback verification step in that case.
