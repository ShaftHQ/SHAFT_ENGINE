In shafthq.github.io package.json the resolutions (yarn) and overrides (npm) blocks must stay semantically in sync, not textually: yarn accepts glob keys like **/sockjs/uuid but npm errors on any such key ('Override without name') and then EVERY npm command fails to parse package.json -- this silently broke the Cloudflare Workers Builds check on PR #828 while yarn CI stayed green. npm scoping syntax is a nested object: "sockjs": {"uuid": "^11.1.1"}.
