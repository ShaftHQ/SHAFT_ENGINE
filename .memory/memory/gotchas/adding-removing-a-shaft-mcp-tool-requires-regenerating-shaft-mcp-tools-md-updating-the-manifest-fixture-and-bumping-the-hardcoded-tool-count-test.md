shaft-skills/references/shaft-mcp-tools.md is a GENERATED catalog (scripts/mcp/generate_shaft_skills_tool_catalog.py) guarded by an installer-verification drift gate in tests/scripts/test_generate_shaft_skills_tool_catalog.py: test_check_mode_passes_against_committed_catalog runs the generator with --check, and test_real_sources_parse_to_expected_magnitude hardcodes the total (e.g. assertEqual(165, total)). So adding/removing an @Tool/@McpTool requires: (1) re-run the generator and commit the result -- NEVER hand-edit the .md (a hand-edit that misses the 'Total tools: N' header passes local Java tests but fails installer-verification on all 3 OS); (2) bump the hardcoded count in that Python test; (3) add the tool to shaft-mcp/src/test/resources/fixtures/mcp-tool-manifest.json -- ShaftMcpApplicationTests.contextRegistersExpectedLeanMcpToolApi() does assertEqual(expected, registeredToolNames) so a missing manifest entry fails the build. Adding only a PARAMETER to an existing tool changes NONE of these (name/description/count unchanged). Evidence: PR #3582 (mobile_api_record_transactions) failed installer-verification until the catalog was regenerated and the count bumped 164->165.
