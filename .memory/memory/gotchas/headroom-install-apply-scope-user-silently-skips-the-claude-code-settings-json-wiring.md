headroom install apply --target claude ... --scope user reports SUCCESS and creates the Windows scheduled tasks, but does NOT merge env.ANTHROPIC_BASE_URL into ~/.claude/settings.json -- verified by reading the file before/after and seeing no change. Root cause in headroom/providers/claude/install.py: apply_provider_scope() (the function that does the settings.json merge) only runs when manifest.scope equals ConfigScope.PROVIDER. Must pass --scope provider, not --scope user, for the actual Claude Code routing to take effect. Re-running headroom install apply with the same profile name and --scope provider updates the existing deployment in place and does perform the merge.
