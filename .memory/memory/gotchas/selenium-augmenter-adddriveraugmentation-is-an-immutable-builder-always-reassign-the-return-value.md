org.openqa.selenium.remote.Augmenter#addDriverAugmentation returns a NEW Augmenter instead of mutating `this` (verified via javap on selenium 4.46.0). DriverFactoryHelper.connectToRemoteServer called it without reassigning, so AddHasCdp/BiDiProvider augmentations were silently no-ops for every remote session until fixed in #3732. Pattern: `augmenter = augmenter.addDriverAugmentation(...)`, never a bare statement.
