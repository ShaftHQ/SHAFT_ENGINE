Chrome 143+ removed 'Pixel 5' (SHAFT's long-standing default mobileEmulationDeviceName) from the DevTools emulated-device list, so every Chrome mobile-emulation session failed with SessionNotCreatedException 'invalid argument: entry 0 of firstMatch is invalid'. Fixed in PR #3421 by extending the previously Edge-only deviceMetrics fallback in shaft-engine OptionsManager to all Chromium browsers (393x851 @2.75 + Pixel 5 UA). The same failure will recur for any other deviceName Chromium drops; issue #3422 item 10 tracks generic validation. Diagnosis shortcut: probe mobile_initialize_web_emulation with a different deviceName (e.g. 'Pixel 7') - if that works, it is device-list drift, not capabilities plumbing.
