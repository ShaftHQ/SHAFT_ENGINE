A minimal consumer pom that only declares the io.github.shafthq:shaft-engine dependency makes Maven Surefire auto-detect the JUnitPlatformProvider (SHAFT ships JUnit Platform 6 transitively), which discovers no TestNG classes: the build passes with Total: 0 and the user never learns their tests did not run. org.junit.support:testng-engine 1.0.6 does NOT fix it (NoClassDefFoundError org/junit/platform/engine/OutputDirectoryCreator against platform 6 with default Surefire 3.2.5). The working fix is maven-surefire-plugin 3.5+ with an explicit org.apache.maven.surefire:surefire-testng plugin dependency - exactly what the SHAFT project generator templates ship. Since PR #3428 the engine prints an actionable warning after any Total: 0 execution summary (ReportManagerHelper.logExecutionSummary) and docs/start/installation.mdx documents the required block. Reproduced empirically on 2026-07-11 in an external consumer project.
