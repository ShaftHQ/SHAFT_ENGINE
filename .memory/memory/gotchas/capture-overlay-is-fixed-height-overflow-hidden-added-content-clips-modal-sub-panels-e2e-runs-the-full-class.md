Two linked gotchas from #3536: (1) #shaft-capture-ui is a fixed-height (min(520px,100vh-32)) flex column with overflow:hidden (only the actions list scrolls). Adding vertical content to the non-scrolling stack (a wrapped/taller header, the first-run coach) can push a modal sub-panel (#shaft-capture-stop-confirm, assertion, locator) partly out of view, making its buttons ElementNotInteractable (no JS error). Fix: gate first-run chrome (maybeShowCoach) on a modalSubPanelOpen() check and refresh on dialog open/close. (2) ManagedCaptureRecorderBrowserTest is @Tag('external-e2e'), only runs under -DincludeCaptureBrowserE2E, and the capture-browser-e2e CI job runs the WHOLE class. Running only your new #method+method selection locally misses cross-test regressions; run the full class headless before pushing: mvn -pl shaft-capture -DincludeCaptureBrowserE2E -Dtest=ManagedCaptureRecorderBrowserTest -DheadlessExecution=true.
