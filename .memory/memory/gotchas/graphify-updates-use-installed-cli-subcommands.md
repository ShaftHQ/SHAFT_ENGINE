Use the installed Graphify CLI syntax before trying slash-command forms: `graphify update .` refreshes the code graph without an LLM. If the active checkout contains ignored or generated artifacts such as Allure reports, `target/`, module-local `graphify-out/`, or temporary test output, do not refresh graphify directly in that checkout and stage the broad diff. Instead create a clean temporary worktree from the PR base, copy in only the intended changed files, run `graphify update .` there, then copy back only the tracked root graphify outputs (`graphify-out/graph.json`, `graphify-out/graph.html`, `graphify-out/GRAPH_REPORT.md`, `graphify-out/manifest.json`, and `graphify-out/.graphify_labels.json` when present). Review `git diff --stat` before staging. This keeps local generated reports out of the shared graph while still satisfying the graphify refresh requirement. Full semantic docs, paper, or image refreshes may require an LLM backend key such as `GEMINI_API_KEY` or `GOOGLE_API_KEY`; report that blocker instead of retrying equivalent commands. If Graphify prints the assistant-level `/graphify --update` notice after doc changes, record it once and do not loop on the same CLI command.
