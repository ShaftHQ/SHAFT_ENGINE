ShaftMcpSetupPanel#installRowInspectionToggle (issue #3601 S2) attached a MouseAdapter to a JPanel step-row so clicking a collapsed 'done' row re-expands it, but JPanel is not focusable by default and had no key binding -- a keyboard-only or screen-reader user could never reach the toggle. Fixed in the #3601 a11y-pass commit by: row.setFocusable(true), row.getAccessibleContext().setAccessibleName(...), and binding VK_ENTER/VK_SPACE on row.getInputMap(JComponent.WHEN_FOCUSED) to an AbstractAction calling the same handler the MouseListener calls. Test pattern: ShaftPanelSetupTest already has a shortcutAction(JComponent, condition, keyCode, modifiers) helper for asserting an input-map/action-map binding exists and invoking it (see doneStepRowToggleIsKeyboardReachable). General rule for this codebase: any addMouseListener(...)-driven interaction on a component that is not already a JButton/JCheckBox/etc. needs an explicit focusable + key-binding check during a11y review -- grep addMouseListener/MouseAdapter across the changed files first.
