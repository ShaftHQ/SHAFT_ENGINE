The capture recorder treats navigations within 10s of a recorded interaction as consequences (not steps), never records `historyUpdated` rewrites or subframe navigations, and records back/forward via overlay-reported `user_traversal` signals (popstate + pageshow) that bypass the interaction window. Overlay-announced navigations carry a `clientActionId`; the pipeline appends or attaches that identity to an already-recorded event (2s attach window).
