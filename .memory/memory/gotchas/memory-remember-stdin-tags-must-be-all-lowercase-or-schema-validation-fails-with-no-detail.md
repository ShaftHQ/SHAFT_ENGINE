memory remember --stdin (and --dry-run) rejects any memories[].tags entry containing an uppercase character with a bare 'error: MemorySchemaValidationFailed: Schema validation failed.' and zero further detail -- no line/field pointer, no --json diagnostic. Confirmed by bisection: a payload otherwise identical to a passing minimal example fails once a single tag like 'ShaftAssistantPanel' (mixed case) is present, and passes again once every tag is lowercased (e.g. 'shaftassistantpanel'). Titles and body text are unaffected -- only the tags array. When this bare schema-validation error appears, check tag casing first before assuming the body/title content is the problem.
