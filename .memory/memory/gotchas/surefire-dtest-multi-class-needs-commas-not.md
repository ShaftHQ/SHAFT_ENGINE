`-Dtest=ClassA+ClassB` does NOT run both classes. Surefire's `+` only joins multiple methods within one class pattern (`Class#m1+m2`). A `+`-joined class list resolves as one unmatched pattern, runs zero tests, produces no surefire-reports directory, and still prints BUILD SUCCESS (masked by `failIfNoTests=false` in the pom). Verified via `-X` (`Resolved included and excluded patterns: <whole unsplit string>`) during #3451. Silent zero-test runs look like passes; combined with 'local mvn test never fails the build' this can fully mask a broken change. Fix: use commas -- `-Dtest=ClassA,ClassB,ClassC` -- and always confirm the surefire/testng XML lists the expected test count.
