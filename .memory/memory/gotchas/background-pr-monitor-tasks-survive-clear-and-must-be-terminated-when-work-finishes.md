During a session babysitting PR CI checks (Monitor/background-bash loops polling `gh pr checks` for shafthq.github.io PRs #805-#812), the user ran /clear. /clear only resets the visible conversation transcript - it does not stop background Monitor/bash tasks tied to the underlying session/job. The 8 poll loops kept running unattended, each re-arming and firing a timeout notification into whatever unrelated task the user gave next, discovered while diagnosing an apparently unrelated Remote Control failure report. Fix applied: TaskStop on each orphaned task id found via the job's fan/inFlight state and via incoming task-notifications. Rule going forward: before reporting a task complete/finished, explicitly stop (TaskStop) any background Monitor/bash polling loops (PR-check watchers, log tails, etc.) that were started for that task and are no longer needed - do not leave them running past the point their result was consumed, and do not rely on /clear or session end to clean them up.
