Root pom's jacoco default-prepare-agent writes the agent into the `argLine` property, but shaft-cli/pom.xml hardcodes <argLine>${surefireArgLine}</argLine>, clobbering it — `mvn -pl shaft-cli test` runs green with NO coverage data produced (silent). Workaround without pom edits: pass `-DsurefireArgLine=-javaagent:<jacoco-agent-path>=destfile=<path>` on the CLI. Real fix tracked in issue #3885. Discovered during the #3886 coverage pass, independently reproduced by hostile review.
