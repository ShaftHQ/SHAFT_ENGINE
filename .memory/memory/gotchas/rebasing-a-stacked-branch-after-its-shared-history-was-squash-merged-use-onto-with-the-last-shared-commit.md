When main advances via a SQUASH-merge of a PR whose commits your branch also contains, a plain `git rebase origin/main` replays those already-landed commits and produces spurious add/add and content conflicts on files that are actually converged. Fix: `git rebase --onto origin/main <last-commit-shared-with-the-merged-PR> HEAD` so only the net-new commits replay. Seen 2026-07-21: ChaosEngine/intellij-tool-selector vs squash commit d0fcaf9662 (PR #3881) — plain rebase cascaded conflicts across shaft-mcp/** and shaft-skills/**; --onto with the 16-commit shared tip (6da92496ae) replayed exactly one net-new commit cleanly. After the rewrite, push with --force-with-lease.
