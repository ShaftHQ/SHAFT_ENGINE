shaft-testng-web/shaft-junit-web/shaft-cucumber-web's generated TestClass.searchForQueryAndAssert (and the Cucumber SampleFeatureFile) searches Wikipedia for "Software testing framework" and asserts the first result's title. The bundled simpleJSON.json hardcoded expectedResultTitle="Unit testing", but Wikipedia's live search ranking for that query now returns "Test automation" as the #1 hit (observed 2026-07-19, reproduced deterministically 4/4 local chrome runs after an initial pair of runs that instead hit an unexplained transient NoSuchElementException on #searchInput that did not recur across 6 further attempts including instrumented ones). This was investigated as issue #3788 Defect C (originally attributed to firefox/grid/BiDi); the deterministic failure mode turned out to be sample-test data staleness against a live, uncontrolled external site, not a SHAFT engine or grid defect -- reproduces identically on local chrome with zero grid/Docker involvement. Fixed by updating expectedResultTitle to "Test automation" in the three example projects' simpleJSON.json plus the Cucumber .feature file. Residual risk: Wikipedia's ranking can drift again; a more robust sample would avoid asserting on a specific ranked search result. The intermittent NoSuchElementException variant was not root-caused (all Actions.java instrumentation showed clean single-iteration find+type+click with zero retries) and firefox/grid could not be tested at all (Docker unavailable in the fixing session) -- see gh issue filed for #3788 Defect C follow-up.
