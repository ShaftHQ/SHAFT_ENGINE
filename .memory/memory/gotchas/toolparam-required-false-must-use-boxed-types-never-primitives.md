Spring AI 2.0 MethodToolCallback passes null for an omitted JSON key; Method.invoke with null bound to a primitive parameter throws IllegalArgumentException, so @ToolParam(required=false) on a primitive (e.g. boolean) produces a schema that invites omission and then crashes the tool call. Use the boxed type plus a null-safe resolver, like capture_start's Boolean headless/resolveHeadless precedent (CaptureService.java). Proven empirically via JsonSchemaGenerator + reflection probe during PR #3718. Direct-Java-call unit tests structurally cannot catch this - the failure only exists through JSON dispatch.
