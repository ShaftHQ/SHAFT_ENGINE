Tried com.intellij.icons.AllIcons.Actions.StartDebugger and AllIcons.RunConfigurations.TestState.Run for new Run/Debug row icons in ShaftTestsPanel (issue #3664, PR #3686). Compiled fine and matched the existing gutter-icon convention in ShaftTestLineMarkerContributor, but when rendered through ShaftPluginScreenshotRendererTest -Dshaft.intellij.screenshotDir=... (this module's lightweight Gradle test harness, no full running IDE Application), both icons painted completely blank -- confirmed visually by generating and reading the actual PNG output before and after the fix, not by inspection. Root cause not fully bisected (plausibly AllIcons' platform-bundled SVGs need a live Application/IconLoader context this harness doesn't provide), but the fix is proven: switch to the ShaftIcons.java pattern (IconLoader.getIcon with light/dark variants bundled under shaft-intellij/src/main/resources/icons/actions/, same as RERUN/CLEAR/SEARCH/CHECK) which renders correctly in the same harness. Rule of thumb: any new icon used inside a Swing component that ShaftPluginScreenshotRendererTest exercises (i.e. most of the plugin's tool-window UI) should go through ShaftIcons' bundled-SVG loader, not a bare AllIcons.* reference -- AllIcons is fine for PSI gutter-icon contributors (RunLineMarkerContributor etc.), which run inside a real IDE, but not for this Swing-unit-test-rendered surface.
