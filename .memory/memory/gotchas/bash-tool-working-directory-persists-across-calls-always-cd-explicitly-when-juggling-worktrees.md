The agent harness Bash tool keeps the working directory from the previous call ('Shell cwd was reset' notes are not guaranteed). In a session juggling multiple git worktrees this caused two real mistakes on 2026-07-18: a branch was created in an agent worktree instead of the main tree, and a memory commit landed on the wrong branch (grid-cdp-comment instead of guidance-budget-redesign), requiring a reset --hard + cherry-pick repair. Rule: every Bash call that depends on repo location starts with an explicit `cd <absolute path> &&`, especially git branch/commit/push operations.
