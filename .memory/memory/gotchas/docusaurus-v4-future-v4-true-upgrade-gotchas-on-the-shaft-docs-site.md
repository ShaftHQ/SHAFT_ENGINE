Enabling `future: { v4: true }` on the SHAFT docs site (shafthq.github.io, PR #756) turns on siteStorageNamespacing, fasterByDefault, mdx1CompatDisabledByDefault, and useCssCascadeLayers at once. Gotchas, all surfaced only at `npx docusaurus build`: `fasterByDefault` needs the `@docusaurus/faster` dependency or the build fails (once on, SWC replaces Babel -> delete `babel.config.js`); `mdx1CompatDisabledByDefault` breaks HTML comments in `blog/*.md` (those release posts double as GitHub release bodyFiles where `<!-- -->` is correct GFM) -- keep `markdown.mdx1Compat: { comments: true }`, do NOT bulk-convert; strict MDX rejects bare `<url>`/`<path>`-style placeholder tokens in `.md` prose (unclosed-JSX-tag error) -- escape as `\<url\>` (tokens inside backticks/fences are safe). Verify with `npx docusaurus build` (check the REAL exit code -- `cmd | tail` masks it with tail's 0), plus `npm run typecheck` and `npm run test:homepage`.
