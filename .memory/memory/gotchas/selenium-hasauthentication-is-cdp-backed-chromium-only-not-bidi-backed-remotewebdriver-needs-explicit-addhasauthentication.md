Only ChromiumDriver implements HasAuthentication directly; FirefoxDriver never does. For RemoteWebDriver it is only obtainable via org.openqa.selenium.remote.AddHasAuthentication, which is NOT auto-discovered by ServiceLoader (META-INF/services lists only BiDiProvider + DevToolsProvider for selenium-remote-driver) -- it must be added explicitly to the Augmenter. SHAFT test names saying 'WebdriverBidi' for basic auth are misleading: the mechanism is CDP. Confirmed correct and kept as-is when fixing #3788 (issue #3732 established this).
