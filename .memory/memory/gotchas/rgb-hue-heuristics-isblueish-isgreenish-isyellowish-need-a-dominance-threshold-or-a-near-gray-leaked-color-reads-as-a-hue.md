ToolApprovalPromptPanelTest's isBlueish/isGreenish/isYellowish helpers originally used a plain greater-than comparison between color channels (e.g. color.getBlue() > color.getRed() && color.getBlue() > color.getGreen()). A leaked dark-theme UIManager override such as Button.background 0x45494A (R=69,G=73,B=74 -- a near-gray with channels only 1-5 units apart) satisfies isBlueish's plain-greater-than check and gets misclassified as a real hue, which is exactly what happened to the Deny button in approvalScopeButtonsShareEqualVisualWeightAndAreColorCodedByScopeBreadth when a prior test's L&F install leaked (PR #3781/#3777's finding). Fix (issue #3782): require the dominant channel to exceed the others by a real margin (HUE_DOMINANCE_THRESHOLD = 15) before classifying as a hue. Verified this threshold still correctly classifies every real production color in ToolApprovalPromptPanel#scopeColor (green 0x8FCB8F/0x2E7D32, blue 0x8FB6E8/0x1565C0, yellow 0xE8D26B/0x8D7A1E all clear the threshold by 50+ units) while correctly rejecting the 0x45494A near-gray leak (delta of only 5).
