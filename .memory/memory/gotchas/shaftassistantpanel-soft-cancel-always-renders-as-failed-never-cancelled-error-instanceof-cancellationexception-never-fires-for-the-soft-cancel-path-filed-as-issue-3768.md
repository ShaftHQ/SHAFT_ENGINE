ShaftMcpInvocation.cancel() (soft) never calls future.cancel() at all -- it only runs cancelAction (sets cancellationRequested, calls Process#destroy()) and always returns true. The invocation's future can therefore only ever complete, after a soft cancel, via AssistantLocalAgentRunner.run()'s own throw new CancellationException(...) once cancellationRequested is observed -- which per the paired CompletableFuture-wrapping gotcha always lands as CompletionException(CancellationException) in whenComplete's error param. ShaftAssistantPanel.showAgentResult/showSequenceResult compute `cancelled = error instanceof CancellationException` directly from that error, so cancelled is always false for a soft cancel, and the run falls through to the plain-failure UI path instead of the Cancelled path. The force-Kill path is unaffected in practice only because it has an independent synchronous shortcut (killedLocalAgentStreamToken set at kill time, checked by handleKilledOrStaleAgentStream before showAgentResult's broken check is ever reached) -- soft cancel has no equivalent shortcut. Filed as SHAFT_ENGINE issue #3768; not fixed (out of scope for #3758, which only touched the force-kill race in ShaftMcpInvocation.kill()).
