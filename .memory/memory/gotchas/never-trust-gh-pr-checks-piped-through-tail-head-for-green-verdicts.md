gh pr checks --watch | tail -N masks failures two ways: tail truncates the failing lines out of view, and the pipeline exit code is tail 0, not gh nonzero-on-failure. This caused two non-green merges (#3454, #3459) that broke main. Correct pattern: redirect gh pr checks --watch output to a file, capture gh own exit code, and grep the final output for non-pass lines before merging.
