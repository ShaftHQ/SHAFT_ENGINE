Verified empirically in the shafthq.github.io docs repo (Docusaurus): the classic Markdown/MDX heading-anchor syntax `## Heading {#id}` fails the acorn MDX parser when used inside a plain `.md` file processed as MDX (Docusaurus v3/v4 default). The working form that actually produces a custom anchor id is the comment-style syntax `## Heading {/* #id */}` -- confirmed by inspecting the built HTML output under `onBrokenAnchors: 'throw'` and finding `id=technology` present on the target heading. Two traps this causes: (1) using the classic `{#id}` form breaks the build outright rather than silently degrading; (2) once built, Docusaurus emits UNQUOTED HTML attributes (e.g. `id=technology`, not `id="technology"`), so any grep/regex pattern that requires `id="..."` to verify an anchor exists gives a false negative against the real built HTML -- search for `id=technology` (or a looser `id=\S*technology` pattern), not the quoted form.
