`shaft-intellij` uses the IntelliJ Platform Gradle plugin, which requires Gradle 9+ and works reliably with JDK 21 in Codex. Do not use the container defaults if they are Gradle 8.x or Java 25: Gradle 8 fails the plugin requirement, and Java 25 can fail Kotlin/Gradle setup before tests. Use a JDK 21 `JAVA_HOME` and Gradle 9 explicitly, for example `JAVA_HOME=/tmp/jdk21 PATH=/tmp/jdk21/bin:$PATH /tmp/gradle-9.0.0/bin/gradle ...` after downloading Temurin 21 and Gradle 9.0.0 into `/tmp` when the repo image does not already provide them. For UI PR screenshots, run `ShaftPluginScreenshotRendererTest` with `-Dshaft.intellij.screenshotDir=target/shaft-intellij-screenshots` for local PR evidence, or the checked-in gallery path only when intentionally refreshing docs/catalog screenshots.
