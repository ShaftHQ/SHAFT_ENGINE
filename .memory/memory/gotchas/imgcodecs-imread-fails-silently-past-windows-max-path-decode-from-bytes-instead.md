OpenCV Imgcodecs.imread(String) returns an empty Mat with NO exception when the absolute path exceeds Windows MAX_PATH (260 chars). Worktrees nested under .claude/worktrees/<name>/ add 40+ chars and pushed a real reference-image path to 293 chars, breaking template matching (issue #3382, fixed in PR #3449). Any OpenCV file read in this codebase should use Files.readAllBytes + Imgcodecs.imdecode instead of imread, mirroring how screenshot bytes are decoded.
