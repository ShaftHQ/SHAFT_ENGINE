JaCoCo in shaft-intellij (IntelliJ Platform Gradle plugin 2.x, `instrumentCode = true`) silently reports 0% for every class unless BOTH fixes are applied (PR #3505, `shaft-intellij/build.gradle.kts`): (1) on the `test` task, `configure<JacocoTaskExtension> { isIncludeNoLocationClasses = true; excludes = listOf("jdk.internal.*") }` -- IntelliJ-instrumented classes carry no source-location attributes so JaCoCo discards their probes (a tiny ~45KB test.exec with zero coverage is the symptom); (2) on `jacocoTestReport`, `classDirectories.setFrom(layout.buildDirectory.dir("instrumented/instrumentCode"))` -- exec class ids match the instrumented bytes, not `build/classes`. If shaft-intellij coverage reads 0%, check these two blocks survived; per-class aggregation folds `Foo$Inner` into `Foo`.
