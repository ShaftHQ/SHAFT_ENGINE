JBTextArea.getEmptyText() (StatusText) clips placeholder lines at a fixed internal width regardless of visible component width: observed ~42-char clipping with 2 appendLine lines and ~30-char clipping with 3 lines in BOTH 360px and 860px renders (lines centered then cut). Do not fight it with shorter fragments: paint the placeholder manually (ShaftAssistantPanel.PlaceholderTextArea overrides paintComponent, measures FontMetrics against getWidth(), word-wraps, draws left-aligned) and expose the text via client property 'shaft.prompt.placeholder' for tests instead of getEmptyText().
