verifyPlugin (Gradle IntelliJ Platform plugin) checks the built plugin against multiple target IDE versions (e.g. IU-243.x and IU-262.x); a newer target flagged com.intellij.openapi.application.ReadAction.compute(ThrowableComputable) as DEPRECATED_API_USAGES and failed the build even though compileJava and all tests passed cleanly on the older bundled test IDE (2024.3). Local `./gradlew test` alone will NOT catch this -- only `./gradlew check buildPlugin verifyPlugin` (the CI command) runs the cross-version plugin verifier. Fix: use `ApplicationManager.getApplication().runReadAction((Computable<T>) () -> {...})` instead (matches the existing pattern in AssistantTranscriptView.renderReadAction). Found and fixed in ShaftTestWatchService.resolveChangedClassFqn (#3641, PR #3655) after a first CI run passed tests but failed verifyPlugin.
