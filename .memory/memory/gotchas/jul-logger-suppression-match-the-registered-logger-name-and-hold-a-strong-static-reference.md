Two ways a java.util.logging suppression silently fails, both hit in DriverFactoryHelper: (1) the logger NAME must match the class's actual static LOG field registration (read the source/javap of the class that declares LOG), not the call-site class printed in the console -- Selenium's WebSocket$Listener.onError logs through the OUTER interface's logger 'org.openqa.selenium.remote.http.WebSocket', so targeting 'WebSocket$Listener' suppresses nothing; (2) LogManager holds only WEAK references to non-root loggers, so fire-and-forget Logger.getLogger(name).setLevel(...) is GC-eligible and gets silently reset -- keep a strong static Logger reference. Also: per-call toggle/restore around driver.quit() races the async SelectorManager socket-reset detection; permanent one-time suppression in a static initializer is the correct shape.
