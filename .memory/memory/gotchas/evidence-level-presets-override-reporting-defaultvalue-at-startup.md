`PropertiesHelper.overridePropertiesForEvidenceLevel()` runs in the post-processing pipeline on every engine startup AND whenever `evidenceLevel` is set. The default level `FAILURE_ONLY` (and `BALANCED`/`FAST`) force-set `traceMode`, `traceEnabled`, `diagnosticsBundleEnabled`, screenshot/pageSource/gif/video and log properties via `applyEvidenceLevel(...)`. So changing a `@DefaultValue` in `Reporting.java` is not enough -- the preset overwrites it at startup (found in #3503: a new `shaft.trace.mode=auto` default would have been silently pinned back to `failure`). How to apply: when changing any reporting/visuals default that appears in `applyEvidenceLevel`, also update the preset rows in `PropertiesHelper.overridePropertiesForEvidenceLevel()` and the assertions in `PropertiesHelperEvidenceLevelUnitTest`.
