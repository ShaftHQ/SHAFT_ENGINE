2026-07-21 incident: the orchestrator ran a read-only inspection with `cd <delegate-worktree> && git log`, and the Bash tool's working directory persisted across subsequent tool calls. A later release-branch `git checkout` + `git merge` + `git push` therefore executed INSIDE the delegate's worktree, switching the delegate's HEAD mid-task (its uncommitted WIP survived but landed on the wrong branch), and the orchestrator initially misattributed the collision to the delegate. The remote push content was still correct; the damage was to the delegate's working state and to diagnosis time. Rule: from the orchestrator/main thread, NEVER bare-`cd` into another agent's worktree. Inspect with `git -C <worktree-path> ...` (stateless), and before ANY state-changing git command verify `pwd` and `git rev-parse --show-toplevel` point at the intended checkout. Extends the existing concurrent-agents-shared-checkout HEAD gotchas.
