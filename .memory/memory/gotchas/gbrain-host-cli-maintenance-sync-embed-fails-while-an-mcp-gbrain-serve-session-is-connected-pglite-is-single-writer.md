Running `gbrain sync --skip-failed` from a host shell while a Claude Code session has the gbrain MCP tools connected fails immediately with `GBrain: Timed out waiting for PGLite lock.` (exit 1, no partial writes). The gbrain CLI's own `sync --help` documents this: "PGLite is single-writer: stop 'gbrain serve' before a large sync". This is the actual reason job 35 (embed-backfill, queued 2026-07-16, attempts_started 0) and the 2 unresolved sync_failures from `gbrain doctor` never drain during an agent session -- not a broken worker, but lock contention with the live MCP server. Fix requires stopping `gbrain serve` first, which kills every concurrent session's gbrain MCP tool access (shared local infra) -- out of scope for an agent to do unilaterally (AGENTS.md 'ask before servers/watchers'); needs the user to stop gbrain serve, run `gbrain sync --skip-failed` and/or `gbrain embed --stale` on the host, then restart serve.
