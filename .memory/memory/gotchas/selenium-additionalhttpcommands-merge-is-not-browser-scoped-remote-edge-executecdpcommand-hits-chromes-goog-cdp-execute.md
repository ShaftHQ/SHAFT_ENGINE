Unlike Augmenter providers (per-browser isApplicable, chrome.AddHasCdp -> CHROME::is, edge.AddHasCdp -> EDGE::is), Selenium's AdditionalHttpCommands ServiceLoader merge has no applicability concept: selenium-chrome-driver and selenium-edge-driver both register "executeCdpCommand" with different URL templates and chrome's /goog/cdp/execute wins even for Edge sessions when both jars are on the classpath (always true for SHAFT). On a remote/grid Edge session this yields UnsupportedCommandException: unknown command. Engine-side remedy: every remote CDP call site needs an UnsupportedCommandException fallback to a non-CDP path (see ScreenshotHelper.takeFullPageScreenshotUsingCDP, issues #3794/#2841). Verified in Selenium 4.46.0 sources during the 2026-07-19 grid verification.
