Extends the stage-by-exact-path rule (see concurrent-agents-in-shared-checkout-must-stage-by-exact-path-never-a-or): in a single non-worktree checkout shared by parallel agents, any agent's `git checkout` switches HEAD for everyone — another agent's later commit then lands on the WRONG branch/PR. Observed twice on 2026-07-21 (T4's commit landed on the sweep branch; the CI-fix coder nearly committed onto the selector branch). Rules: (1) immediately before every commit, re-verify `git branch --show-current` and the expected tip; (2) prefer `git worktree add` isolation for any agent that can run concurrently with another; (3) never stash/reset another agent's uncommitted files — hide them from a rebase's clean-tree check with `git update-index --skip-worktree <paths>` and unset afterwards.
