shafthq.github.io scripts/generate-properties-catalog.mjs merges hand-written PropertiesList.mdx content into the generated catalog. Its section-header regex expected '### Name' while the mdx has used '## Name' since the generator was introduced (#819), so mdxLookup was silently empty on every run and all rich hand-written descriptions degraded to humanized/javadoc fallbacks. The generator's --check (regenerate-and-diff) stayed green the whole time because the output was self-consistent — idempotence checks cannot catch losing an input source entirely. Fixed in shafthq.github.io PR #844 ('## ' + pure parseMdxDefaultsTablesFromContent seam). Lesson: when a generator merges a hand-written source, add a regression test asserting one known hand-written value survives regeneration end-to-end.
