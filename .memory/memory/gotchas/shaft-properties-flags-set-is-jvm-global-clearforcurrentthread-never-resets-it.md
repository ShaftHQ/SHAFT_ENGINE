Flags.setProperty (shaft-engine/src/main/java/com/shaft/properties/internal/Flags.java ~18-26) writes via ThreadLocalPropertiesManager.setGlobalProperty into the static JVM-wide globalOverrides map (ThreadLocalPropertiesManager.java ~42-46) and republishes Properties.baseFlags. Other property groups (reporting, web, visuals, ...) use per-thread *Override ThreadLocals that Properties.clearForCurrentThread() removes (Properties.java ~217-236) — the removal list has no flags entry because no flagsOverride ThreadLocal exists. Consequence: any test calling SHAFT.Properties.flags.set().anything(...) that relies on Properties.clearForCurrentThread() in a finally block silently leaks the change to every later test in the JVM run. Correct pattern: capture the original value first and restore it via SHAFT.Properties.flags.set() in finally (or @BeforeMethod/@AfterMethod) — reference implementations: testPackage.unitTests.FlagsSetPropertyCoverageUnitTest and the #3503 retry-aware trace tests in FailureTraceReporterTest (PR #3507).
