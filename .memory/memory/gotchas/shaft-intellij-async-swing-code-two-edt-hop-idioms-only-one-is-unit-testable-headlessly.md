In shaft-intellij's Gradle unit-test JVM, ApplicationManager.getApplication() is null, so any code path that unconditionally calls ApplicationManager.getApplication().invokeLater(...) (e.g. ShaftSettingsConfigurable#testMcpConnection, #runProviderKeyProbe) is NOT directly unit-testable and NPEs if invoked from a test. The alternative idiom - chaining directly off a CompletableFuture returned by ShaftCredentialService's *Async methods (e.g. apiKeyAsync/hasApiKeyAsync/updateStoredStateAsync), with no explicit invokeLater at all - IS unit-testable, because those methods already guarantee landing back on the EDT internally in production, and a test fake's pre-completed future makes the continuation run synchronously inline. When adding a new async Swing flow in this module, use the *Async-chaining idiom for any step that only needs a credential/state lookup, and reserve the unconditional-invokeLater idiom (matching existing convention, not a new guard) for the step that does the real blocking work (network call, MCP probe) via ShaftPluginExecutor.getInstance().executor() - then test the blocking step with a bounded CountDownLatch.await(N, SECONDS), never a sleep loop.
