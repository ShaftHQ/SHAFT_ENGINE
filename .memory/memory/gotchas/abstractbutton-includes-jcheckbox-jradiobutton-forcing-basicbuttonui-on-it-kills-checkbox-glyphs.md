In shaft-intellij's ShaftPluginScreenshotRendererTest.applyComponentTheme(), `if (component instanceof AbstractButton button)` unintentionally also matches JCheckBox and JRadioButton (both are AbstractButton via JToggleButton). Forcing `button.setUI(new BasicButtonUI())` on those (originally added in PR #3157 to fix plain JButton backgrounds under headless light IntelliJLaf) replaces the L&F's own CheckBoxUI/RadioButtonUI, which is what supplies the checkbox glyph via getDefaultIcon(). BasicButtonUI's property prefix is "Button.", not "CheckBox.", so button.getIcon() stays null and nothing paints -- producing a flat filled block instead of a checkbox glyph, identical regardless of selected state. Fix: guard the UI swap with `!(button instanceof JToggleButton)` so JCheckBox/JRadioButton keep their L&F UI delegate. Filed/fixed as issue #3777, PR #3781.
