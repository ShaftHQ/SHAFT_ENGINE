sikulixapi 2.0.5 bundles a manifest (sikulixlibs per-os libs64 sikulixcontent) listing opencv_java430.dll, but org.openpnp:opencv:4.9.0-0 (pinned by shaft-sikulix since #3408) only ships opencv_java490.dll. SikuliX libsExport fails the copy, deletes the whole SikulixLibs folder, marks itself done, and never retries; every loadLibrary then throws 'not in any libs folder'. SHAFT pre-stages the correct native plus version marker via SikuliNativeLibraryStager (#3805). If the openpnp pin is ever bumped, the stager keys off Core.NATIVE_LIBRARY_NAME so it follows automatically.
