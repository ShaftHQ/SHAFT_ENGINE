headroom (github.com/chopratejas/headroom) was adopted 2026-07-16 (PRs #3616/#3617) as a local context-compression proxy fronting Claude Code via env.ANTHROPIC_BASE_URL=http://127.0.0.1:8787 in the global ~/.claude/settings.json, and fully removed 2026-07-17 by owner decision. While it genuinely reduces token usage, in practice it (1) blocks remote control of Claude Code sessions, (2) reduces the effective context window from 1M to 200K tokens, and (3) breaks automatically loaded skills/instructions -- net effect is a materially dumber agent. Do NOT re-adopt headroom (or any similar API-fronting compression proxy) unless all three problems are demonstrably solved. Removal gotchas: under the persistent-docker preset the pip CLI (headroom-ai) is not installed, so the documented 'headroom install unwrap claude' revert path does not exist -- manual revert is: remove env.ANTHROPIC_BASE_URL from ~/.claude/settings.json (preserving unrelated env keys such as ENABLE_TOOL_SEARCH), docker rm -f headroom-default (its restart policy is unless-stopped, so a plain 'docker stop' resurrects it on reboot), remove the ghcr.io/chopratejas/headroom image, and delete the ~/.headroom state dir plus ~/.claude/settings.json.pre-headroom.bak. All five prior headroom memory objects (2 decisions, 2 facts, 1 install gotcha) were deleted along with the AGENTS.md/CONTRIBUTING.md/docs-site runbook references.
