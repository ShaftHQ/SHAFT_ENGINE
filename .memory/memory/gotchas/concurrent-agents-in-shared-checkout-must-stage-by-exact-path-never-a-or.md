When multiple agents work concurrently on the same branch in a shared git checkout (not worktree-isolated), they must stage changes with extreme care. Using broad git add -A or git add . can accidentally sweep up ANOTHER concurrent agent's in-flight, uncommitted files, corrupting or misattributing their work. This happened independently to three concurrent agents on branch ChaosEngine/tool-architecture-sweep this session. Mitigation: always run git status and git diff --stat before staging to confirm what's yours, then stage only the exact explicit paths you own — never broad flags — when you know or suspect other agents may be concurrently active in the same checkout. If unsure, use worktree isolation (git worktree add) instead of a shared checkout for concurrent dispatch.
