From PR #3490: Marketplace plugin 10.3.20260712 was uninstallable ('Failed to load the plugin descriptor') -- a `--` inside an XML comment in the optional `shaft-withJUnit.xml` descriptor made IntelliJ reject the whole archive; the fix must ship under the NEXT date-based `pluginVersion` (Marketplace versions are single-use; the release flow owns the bump in `shaft-intellij/gradle.properties`). `verifyPlugin` default `failureLevel` is COMPATIBILITY_PROBLEMS only, so deprecated APIs and structure defects were warnings and CI passed while Marketplace flagged them -- strict failure levels + `-Xlint:deprecation -Werror` are now set in `shaft-intellij/build.gradle.kts` (all four workflows inherit). Marketplace gallery images are a MANUAL upload (no API); renderer shots go to `marketplace-media/`, regenerate via `ShaftPluginScreenshotRendererTest -Dshaft.intellij.screenshotDir=...` (output 860x780). Local `verifyPlugin` needs JDK 21 Gradle (~8 min after a ~1 GB first-time IDE download); reports land in `shaft-intellij/build/reports/pluginVerifier/`.
