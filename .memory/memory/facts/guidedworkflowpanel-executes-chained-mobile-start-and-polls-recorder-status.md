Since PR #3424 the Guided panel is not purely prefill-only: with an MCP-connected project, Start recording on the Mobile backend runs mobile_initialize_web_emulation then mobile_record_start directly (gated on success), and every backend's Start begins polling capture_status / mobile_record_status / playwright_record_status into a Status strip. With a null project (unit tests, GuidedWorkflowLiveE2ETest PanelDriver) behavior stays prefill-only, which the live E2E harness depends on. The panel implements Disposable; ShaftToolWindowPanel disposes it on view switches.
