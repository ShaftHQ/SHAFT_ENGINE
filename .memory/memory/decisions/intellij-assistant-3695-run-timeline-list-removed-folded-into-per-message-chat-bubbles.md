The Assistant panel's separately-scrollable 'Run timeline' JList (timelineModel/timeline/timelinePanel, TimelineListCellRenderer) is removed entirely. Every signal that used to feed it (tool selection, 'Running', streamed notifications/progress milestones via onToolProgress, terminal Completed/Failed/Cancelled/Killed via appendTerminalAgentMilestone, non-verbose compact local-agent output via addCompactLocalAgentMilestone, 'Auto-approved SHAFT tool: X', 'Cancelling...'/'Killing...') now renders as its own persisted chat bubble in the main transcript via ShaftAssistantPanel#appendAgentMilestone -> append('assistant', ...), reusing the existing bubble mechanism (no new rendering path). Bubble text is prefixed with the same success/error/warning/pending glyph the old TimelineListCellRenderer used (ShaftAssistantPanel#agentMilestoneIcon), preserving that at-a-glance signal. The NON_ACTIONABLE_META_TOOL_NAMES (ToolSearch) filter from issue #3672 is unchanged and still applies before a milestone becomes a bubble. Below the chat, the single-line status JLabel + JProgressBar (ShaftAssistantPanel#status/#progress, transcriptStatus panel) is unchanged and is now the only run-status element left below the composer -- it already had no 'Run timeline'-style label. See the companion gotcha memory about the streaming-placeholder replace-last regression this change required fixing.
