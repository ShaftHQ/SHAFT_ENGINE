Evaluated a real Obsidian vault + Obsidian MCP as a gbrain replacement (user hard reqs: fully local + source-controllable). Verdict: no migration. Findings 2026-07-18: no official Obsidian MCP exists. Community servers split two ways. (1) REST-API servers (MarkusPfundstein/mcp-obsidian ~4.1k, cyanheads ~633) require the Obsidian DESKTOP GUI app running plus the Local REST API plugin -- a live Electron GUI process, which violates AGENTS.md No GUI/shell-open and is hostile to headless/CI; rejected. (2) File-based servers (bitbonsai/mcpvault ~1.5k, marcelmarais, Anthropic Filesystem MCP) read .md off disk with no app/plugin/proxy and satisfy both hard reqs, but are just plain markdown + wikilinks + a thin CRUD/search wrapper -- functionally what the memory CLI already provides over .memory/, with weak/absent backlink-graph and no semantic/embedding retrieval, so they do not replace gbrain value either. None require ANTHROPIC_BASE_URL/proxy. Conclusion: the already-adopted .memory/ plain-markdown + [[wikilinks]] + MEMORY.md MOC IS the fully-local, git-source-controllable store the user wants (zero runtime dependency -- reinforced by gbrain fragility); a file-based Obsidian MCP would be redundant tooling (ponytail duplication), and the Obsidian app is rejected outright. Upholds and extends the 2026-07-12 verdict against the 44-command obsidian-second-brain suite; gbrain recovered retrieval removes any remaining pressure to migrate.
