A new check_release_needed job runs check_maven_release_version.py --github-output "$GITHUB_OUTPUT" first and outputs already_released. build_release_and_deliver, verify_public_shaft_mcp_installer, and announce_release all gate on this (build_release_and_deliver via needs.check_release_needed.outputs.already_released != 'true'; the other two via needs.build_release_and_deliver.result == 'success') so a push to main between releases shows a clean skip instead of a hard failure. The Codecov upload step also gates on steps.pilot_tests.outcome != 'skipped' so an earlier failed/skipped build step doesn't add a second, misleading 'JaCoCo report missing' failure. check_maven_release_version.py keeps its old hard-fail (exit 1, no --github-output) behavior for prepare-release-pr.yml, where an already-existing version during release prep is a genuine guard violation, not a no-op.
