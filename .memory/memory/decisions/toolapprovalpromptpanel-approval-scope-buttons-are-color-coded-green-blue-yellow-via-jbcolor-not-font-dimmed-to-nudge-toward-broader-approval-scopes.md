Issue #3696: the prior implementation shrank the font and grayed the foreground of 'Approve tool always' and 'Approve all tools' (the broader-trust scopes) while leaving 'Approve once' at full default emphasis -- the opposite of the intended nudge, since a dimmed/unbordered button reads as 'avoid this'. Replaced with `applyScopeEmphasis`/`scopeColor` in ToolApprovalPromptPanel.decisionButton(): all three scope buttons keep equal font size and get an opaque, bordered fill via JBColor(light, dark) pairs -- green 0x8FCB8F/0x2E7D32 for APPROVE_ALL_TOOLS (broadest), blue 0x8FB6E8/0x1565C0 for APPROVE_TOOL_ALWAYS (medium), yellow 0xE8D26B/0x8D7A1E for APPROVE_ONCE (narrowest) -- with a single `new JBColor(Color.BLACK, Color.WHITE)` foreground pair (chosen because every light-theme fill is a light/medium pastel and every dark-theme fill is a deep, saturated shade, so one black/white pair covers both without per-color contrast math). DENY intentionally keeps the platform default look; it is not a scope to weight. Verified in both IntelliJ themes via ShaftPluginScreenshotRendererTest's renderApprovalPrompt(...) (light committed to the existing gallery; dark verified manually via reflection, not committed as a new gallery asset).
