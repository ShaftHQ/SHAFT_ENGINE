Field failures from a fresh external project and durable fixes (PR #3431, docs PR shafthq.github.io#736): (1) Recordings must never belong to a one-shot agent turn: 'Record my browser actions on https://...' fell through AssistantCommand's intent matchers to the local agent CLI, which ran capture_start inside its own ephemeral shaft-mcp child; the recording died seconds later when the turn ended (INCOMPLETE, 3 events). Fixes: isBrowserRecordingIntent/isMobileRecordingStartIntent match plain-language 'record ...' phrasings including every plugin prefill; mobile recording intents are checked BEFORE isMobileControlIntent (which had swallowed 'Record my mobile actions on the Android emulator' into mobile_toolchain_status); the local-agent preamble forbids interactive recordings from one-shot turns while same-turn scripted capture_start_codegen stays allowed. (2) Capture lifecycle must self-heal, never wedge: the active session JSON vanished mid-run (Windows non-atomic replace-move under file-watcher/AV contention); CaptureEventPipeline.eventCount() -> store.read() threw 'Capture session has not been started.' from inside status(), wedging CaptureManager permanently ('already active' vs 'not started' split-brain until process death). Fixes: CaptureSessionStore keeps a last-persisted snapshot and republishes it when the file vanishes, store.discard() clears the snapshot so intentional discards stay discarded, ManagedCaptureRecorder.status() never throws, CaptureManager.stop() releases the session even when recorder stop throws, CaptureManager.start() verifies an active-looking lastStatus against the live recorder before rejecting, and CaptureJsonCodec.atomicWrite publishes content directly when the replace-move fails. (3) Element tools bridge to the active capture session: element_*/natural_act only used EngineService's driver_initialize session, making the documented agent-performed codegen flow (capture_start_codegen -> element_type -> capture_stop -> generate) impossible ('No active browser session'). Fix: CaptureService's Spring constructor registers a bridge and EngineService.getDriver() falls back to CaptureManager.activeShaftDriver(); driven interactions are recorded like user ones (verified live, events 2->5). (4) FileActions folder+file joins insert a separator; a folder property without a trailing slash had produced allure-resultsenvironment.xml at every consumer project root; shaft-mcp MCP_PATH_PROPERTIES keep trailing slashes.
