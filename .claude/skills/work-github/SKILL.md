# Work GitHub

Read `references/playbook.md` in full before executing any phase — it is
short and covers: grounding scope before asking, the one upfront
AskUserQuestion for branch/PR/merge-authority, task tracking, risk-first
sequential-vs-parallel subagent dispatch (every subagent loads
`act-as-fable` too), reviewing a subagent's diff before committing it,
scoping docs/screenshot updates to real changes, the Learning Loop
(memory/issues/skills) before wrap-up, and the push/PR/green/merge/compact
close-out.
